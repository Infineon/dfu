Field Documentation

◆ dataBuffer

uint8_t* cy_stc_dfu_params_t::dataBuffer

The pointer to a buffer that keeps data to read or write to an NVM.

It is required to be 4-byte aligned.

◆ packetBuffer

uint8_t* cy_stc_dfu_params_t::packetBuffer

The pointer to a buffer that keeps packets sent and received with the Transport API.

It is required to be 4-byte aligned.

◆ timeout

uint32_t cy_stc_dfu_params_t::timeout

The time (in milliseconds) for which the communication interface waits to receive a new data packet from Host in Cy_DFU_Continue().

A typical value is 20 ms.

◆ appId

uint32_t cy_stc_dfu_params_t::appId

Set with the Set App Metadata DFU command.

Used to determine an appId of a DFU image

◆ initCtl

uint32_t cy_stc_dfu_params_t::initCtl

The initial value to the ctl parameter for Cy_DFU_ReadData and Cy_DFU_WriteData.

The DFU SDK functions call the Read/Write Data functions like this:
Cy_DFU_ReadData(addr, length, CY_DFU_IOCTL_COMPARE, params).
